在Go语言学习中遇到到了瓶颈

关注了两年的Go语言，最近一年才真的着手去编程写Go(课太多我也不想).. Go学起来不难，但是还是总结一下Go的软件工程概念和编程哲学（感谢七牛云许大的书，写得真的不错，初学用的英文文档，上手很快但对于一些概念不懂，看了这本书理解了很多，虽然才看了一部分）

Go在软件工程中的优势

代码风格规范

和python差不多，强调代码规范，如强调要求public的变量开头字母大写（以前写代码老忘记为何报错 = = ，老错在这里），private的变量小写字母开头，其使代码简洁且命名规范。另外Go对花括号进行了强制，这种强制当时不理解，但是代码写长了我发现还这样真棒。。。ε=ε=ε=(~￣▽￣)~

defer关键字

这个设计真的很棒，以前学JAVA网络编程老爱忘记socket.close(),各种try,finally看的眼花缭乱，这个收尾关键字在处理错误时体现了很好的优势。

Go的编程哲学

要说Go是面向对象还是面向过程我真的说不清楚，学了这一段时间，我更觉得Go是去面向处理问题的编程语言，以下的话来自于Go语言编程：

Go对于面向过程和面向对象编程，做出了相应的融合，去除复杂化，极力去支持语言的简洁性和可读性。

Go反对函数和操作符的重载（overload）..Go语言支持类、类成员方法、类的组合，但反对继承、反对虚函数和虚函数重载，既然Go放弃了虚函数，自然没有了构造函数和析构函数。

之所以说Go简单，它将许多OOP繁杂的概念避开了，既然没有这些特性，但Go提供了一个很好的特性：接口（interface）,我至今在看接口时还是不太理解，这个可能是Go语言设计中及其巧妙的地方，我还是多钻研下，GopherChina2017的Google大佬的演讲我再去看看。

Go语言非侵入接口的设计使得标准库不需要绘制类库的继承树图，只需要知道这个类实现了哪些方法，方法具体意义即可。另外不用再纠结接口需要拆得多合理（深有体会，java的文件类派生就是个典例）

Go对并发的支持和一些编程思想吸收

Go接受了部分函数式编程的想法，支持函数与闭包。在并发方面，接受了Erlang的面向消息编程思想，支持goroutine和通道，强调通过消息来共享内存来进行并发。

总结

自己还在翻来覆去啃思想和一些实战的过程中，多学，多敲代码多练，多写笔记。即使Go不招应届生，我选择Go，我就不放弃Go的学习。
